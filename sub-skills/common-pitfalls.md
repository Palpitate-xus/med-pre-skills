# Sub-Skill: 常见陷阱与规避方法

> **所属 Meta-Skill:** [临床预测模型开发指南](../clinical-prediction-model-development-zh.md)

---

## 方法学陷阱

### 1. 将连续型预测变量二分类化

| 方面 | 说明 |
|------|------|
| **问题** | 丢失信息，引入任意阈值，扭曲剂量-反应关系 |
| **错误示例** | 年龄分 "≥65 岁 / <65 岁" |
| **正确做法** | 使用限制性立方样条（`cr()`）或 GAM（`s()`） |
| **后果** | 统计功效下降，校准变差，可能错误估计风险 |

### 2. 使用逐步回归

| 方面 | 说明 |
|------|------|
| **问题** | 增加偏倚，导致过拟合，系数估计偏倚 |
| **错误示例** | `step(lm(y ~ ., data = data))` |
| **正确做法** | 岭回归 / LASSO / Elastic Net |
| **后果** | 表观性能虚高，泛化能力差 |

### 3. 单变量 P 值筛选预测变量

| 方面 | 说明 |
|------|------|
| **问题** | 多重检验问题，忽略变量间相关，增加偏倚 |
| **错误示例** | 只纳入单变量 P < 0.05 的变量做多变量模型 |
| **正确做法** | 基于先验知识预指定变量集，用惩罚控制复杂度 |
| **后果** | 遗漏重要但"不显著"的变量，纳入假阳性变量 |

### 4. 将连续型结局二分类化

| 方面 | 说明 |
|------|------|
| **问题** | 损失统计信息，引入任意阈值 |
| **错误示例** | 将 mRS 0-6 拆成 "良好 0-2 / 不良 3-6" |
| **正确做法** | 有序 Logistic 回归，或保留连续型分析 |
| **后果** | 信息损失 30-50%，统计功效大幅下降 |

## 验证陷阱

### 5. 仅报告表观性能

| 方面 | 说明 |
|------|------|
| **问题** | 训练集上的性能永远过于乐观 |
| **错误示例** | "模型 AUC = 0.90"（未说明是表观还是校正） |
| **正确做法** | 报告 Bootstrap 乐观校正后的 AUC |
| **后果** | 发表后外部验证时性能断崖式下跌 |

### 6. 使用简单拆分样本验证

| 方面 | 说明 |
|------|------|
| **问题** | 浪费数据，小样本中极不稳定 |
| **错误示例** | 70/30 或 80/20 随机拆分 |
| **正确做法** | Bootstrap（200-500 次）或 K-fold CV |
| **后果** | 验证结果不可靠，尤其小样本 |

### 7. 忽视校准度

| 方面 | 说明 |
|------|------|
| **问题** | 仅有区分度不足以支持临床决策 |
| **错误示例** | 只报告 AUC，不报告校准图 |
| **正确做法** | 报告校准截距、斜率、校准图 |
| **后果** | 模型给出的风险值不可信 |

## 数据陷阱

### 8. 完全案例分析

| 方面 | 说明 |
|------|------|
| **问题** | 损失样本，若缺失非 MCAR 则引入偏倚 |
| **错误示例** | `data.dropna()` 后直接分析 |
| **正确做法** | 多重插补（`miceforest` / `sklearn IterativeImputer`） |
| **后果** | 样本代表性下降，估计偏倚 |

### 9. 小样本 / 多参数

| 方面 | 说明 |
|------|------|
| **问题** | 过拟合风险极高 |
| **错误示例** | N=200，纳入 30 个参数 |
| **正确做法** | 先计算样本量，控制参数数量 |
| **后果** | 模型完全记住训练数据，外部验证失败 |

### 10. 忽视临床效用

| 方面 | 说明 |
|------|------|
| **问题** | 统计性能好 ≠ 临床有用 |
| **错误示例** | AUC 0.85 + 校准好，但未评估是否改善决策 |
| **正确做法** | 进行决策曲线分析（DCA） |
| **后果** | 开发了临床医生不会用的模型 |

## 快速自检表

在开始分析前，逐条确认：

- [ ] 连续变量保持连续（用样条）
- [ ] 未使用逐步回归或单变量 P 值筛选
- [ ] 样本量计算已完成，参数数在预算内
- [ ] 使用多重插补处理缺失数据
- [ ] 使用了惩罚/收缩方法
- [ ] 将进行 Bootstrap 乐观校正
- [ ] 将评估校准度（截距 + 斜率 + 图）
- [ ] 将进行决策曲线分析
- [ ] 将按 TRIPOD 报告并分享代码
